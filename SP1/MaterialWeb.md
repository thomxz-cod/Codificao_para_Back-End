## 🔍 Levantamento de Requisitos

É o processo de entender **o que o sistema precisa fazer**, baseado nas necessidades do cliente.

### 📌 Tipos de Requisitos
- **Funcionais (RF):** o que o sistema faz  
- **Não Funcionais (RNF):** como o sistema funciona (desempenho, segurança, etc.)

---

## 🔄 Front-End vs Back-End

- **Front-End:** interface e interação com o usuário  
- **Back-End:** regras de negócio e processamento de dados  

---

## 🛠️ Técnicas de Levantamento de Requisitos

### 📋 Briefing
Coleta inicial de informações com o cliente (objetivos, público, prazo, etc.)

### 💡 Brainstorming
Geração de ideias em grupo, sem julgamentos

### 📝 Questionário
Lista estruturada de perguntas para usuários

### 🗣️ Entrevista
Coleta de feedback após o sistema já existir

### 👀 Etnografia
Observação do comportamento dos usuários

### 🧪 Prototipagem
Criação de modelos visuais antes do desenvolvimento

---

## 📈 Importância

Essas técnicas ajudam a:
- Reduzir erros  
- Melhorar a comunicação com o cliente  
- Definir melhor o escopo do projeto  
- Aumentar a qualidade da entrega  

---

## ✅ Conclusão

O levantamento de requisitos é essencial para o sucesso de qualquer sistema.  
Ele garante que o produto final atenda às necessidades do cliente com eficiência e qualidade.
